Measuring distance

  • Generalized distance: Lp metrics for measuring distance on meshes or grids
  • distance between $a(x_1,y_1)$ and $b(x_2,y_2)$
  • $dp(a,b)=[(x_1-x_2)^p + (y_1-y_2)^p]i^{1/p}$
  • p=2 : Euclidean distance
  • p=1 : Manhattan (Taxicab, City block) metric

  • Linear Referencing Systems: location is defined by its linear distance from a known point on a route (e.g. milepost on a highway), and distance is measured along segments of a network; used for roads, utilities, waterways

Raster map buffer operations

Proximity analysis function
  • discrete:
    • creates buffer zones (equal distance zones) from a given feature (point, line or area)
    • applies to raster map layers representing discrete objects
    • resolution should be sufficiently higher (cell size should be n-times smaller) than distances (buffer widths)
  • continuous:
    • derives proximity raster from a given feature
    • computes distances from a feature for every cell, "growing" in perpendicular direction
    • resolution controls the distance precision - step

    Cost surfaces

    Cumulative (accumulated) cost surface:
    • raster representation of cumulative cost of reaching the target point(s) from any location in the given region.
    Input cost map
    • value assigned to a cell represents cost of traversing that cell,
    • cost can be based on time, energy, etc.

    Cumulative cost surface to a point

    Proximity surface representing shortest path (distance) from each cell to the target point if cost for each cell is 1:

    Cumulative cost surface

    cost for each cell is 1

    Cost map

    If cost is spatially variable, we need to define cost to traverse each grid cell, usually expressed as time $t[s]$ derived from:
    • inverse of speed limit (or average speed data) for driving,
    • slope and vegetation density for walking on trails

    Example: cost surface derived as minimum time $t[s]$ needed to traverse a grid cell with resolution d[m] $$ t = d / v $$ where, v is speed (e..g speed limit stored in roads/streets data layer. Cost map is computed using map algebra.

    Cost map

    Deriving a cost map from speed limit, as time to traverse a 30m cell [seconds] off-road cells are assigned to 5 mi/hr, for realistic travel time per cell speed limit [mi/hr time to traverse a 30m cell [seconds] Cost can be defined par grid cell or per map unit, depending on implementation

    Cumulative cost surface

    Cumulative cost surface: time to reach the point A, the higher the hill the more time it takes to reach A Application: locate the firestation that can reach the accident the fastest and identify the shortest path
